## Option B Overview

### Concept
A classic corporate website structure with dedicated pages for each major section. Clear navigation hierarchy with dropdown menus for subsections.

### Strengths
- **SEO-Optimized**: Separate pages allow granular keyword targeting
- **Content Depth**: Each page can go deep into specific topics
- **Familiarity**: Users understand traditional navigation patterns
- **Analytics**: Easy to track page performance and conversion paths
- **Scalability**: Easy to add new pages without restructuring

### Weaknesses
- **More Clicks**: Users need to navigate multiple pages to find information
- **Page Load**: Each navigation requires new page load (even if fast)
- **Fragmentation**: Story can feel disconnected across pages
- **Less Modern**: Traditional pattern feels less cutting-edge

### Best For
- Companies prioritizing SEO and organic search
- Brands with deep content libraries
- Audiences expecting traditional navigation
- Long-term content strategy and growth

### Reference Inspiration
- **Petro West**: Clear service pillars with dedicated pages
- **Kinley Construction**: Division structure, project showcase
- **Guardian Fueling**: Professional multi-page corporate site

## Component Specifications

### Breadcrumb Navigation

**Purpose:** Show user location in site hierarchy
**Placement:** Below header on all non-homepage pages
**Format:** Home > Section > Current Page
**Behavior:**
- Each level is clickable link except current page
- Current page is plain text (not linked)
- Separator: > or / character
- Mobile: May abbreviate or truncate on narrow screens

### Dropdown Menus

**Desktop Behavior:**
- Hover to open (no click required)
- 0.2s delay before closing (prevents accidental close)
- Keyboard accessible (Tab to navigate, Enter to select)
- Visual indicator: Chevron icon on hover items

**Mobile Behavior:**
- Tap to expand/collapse accordion-style
- Chevron rotates 180Â° when expanded
- Nested items indent for visual hierarchy

### Page Transitions

**Between Pages:**
- Standard page load (no AJAX)
- Optional: Fade-in animation on new page load (0.3s)
- Loading indicator if page takes > 1s to load
- Scroll position resets to top on new page

**Within Page:**
- Smooth scroll for anchor links
- Accordion expand/collapse animations (0.3s ease)

### Form Specifications

**Contact Forms (on each page):**
- Simplified forms compared to dedicated Contact page
- Fields: Name, Email, Phone, Message
- Context-aware: Market/Service pre-selected based on page
- Submission routes to appropriate branch

**Quote Request Forms:**
- Multi-step on Contact page
- Single-page simplified version on market/service pages
- Market/Service field auto-populated from page context

## Performance Considerations

### Page Load Optimization
- Critical CSS inlined for above-fold content
- Lazy load images below fold
- Minify CSS/JS files
- Compress images (WebP with JPG fallback)
- Target: < 3 seconds initial load on 3G

### Code Splitting
- Separate CSS files per page template (homepage.css, market.css, service.css)
- Load only CSS/JS needed for current page
- Defer non-critical JavaScript

### Caching Strategy
- Browser caching for static assets (CSS, JS, images)
- CDN for static assets (optional)
- Server-side caching for page HTML (5-60 minutes)

**Version:** 1.0

## Option A Overview

### Concept
A modern single-page scrolling experience with anchor navigation. All major content on one continuous page with smooth scroll to sections.

### Strengths
- **Storytelling Flow**: Natural narrative from hero â†’ markets â†’ services â†’ why us â†’ contact
- **Mobile-Friendly**: Eliminates multi-page navigation complexity on mobile
- **Modern Feel**: Contemporary web design pattern, feels fresh and bold
- **Fast Loading**: One initial page load, then instant anchor jumps
- **Memorable**: Users experience complete story in one session

### Weaknesses
- **SEO Challenges**: Single URL means less granular page optimization
- **Content Depth**: Limited space for detailed service/market information
- **Analytics**: Harder to track which sections drive conversions
- **Load Time**: Initial page load slightly heavier (all content at once)
- **Maintenance**: Harder to update individual sections

### Best For
- Companies wanting bold, modern statement
- Brands with clear, concise messaging
- Mobile-first audiences
- Memorable first impressions

## Component Specifications

### Sticky Header Behavior

**Desktop:**
- Fixed position at top of viewport
- Background: White with 1px bottom border (Neutral 100)
- Height: 80px
- Sticky on scroll (follows user down page)
- Z-index: 1000

**Anchor Navigation Bar:**
- Appears after scrolling past hero section
- Background: White with subtle shadow
- Highlights current section with blue underline
- Smooth scroll to section on click
- Height: 60px
- Z-index: 999 (below main header)

**Mobile:**
- Fixed position
- Height: 60px
- Hamburger menu opens full-screen overlay
- Service button opens modal
- Call button triggers tel: link

### Section Transitions

**Scroll Behavior:**
- Smooth scroll with easing (CSS: `scroll-behavior: smooth`)
- Fade-in animations on scroll into viewport
- Parallax effect on hero background (subtle)

**Anchor Links:**
- URL updates with section ID (e.g., `#markets`, `#services`)
- Browser back button navigates between sections
- Shareable URLs to specific sections

### Interactive Elements

**Markets Cards:**
- Hover: Subtle lift effect (transform: translateY(-8px))
- Hover: Box shadow increases
- Click: Expands to modal overlay with more market details
- Modal: Includes subsections, case studies, CTA to request quote

**Lifecycle Wheel:**
- Each service icon is clickable
- Click: Modal overlay with service details
- Modal: Includes description, process, related services, CTA

**Map:**
- Interactive (click branches for details)
- Zoom in/out controls
- Hover: Highlight branch pin
- Click: Shows branch address, phone, services

### Form Specifications

**Contact Form Fields:**
- Name (text input, required)
- Email (email input, required, validation)
- Phone (tel input, required, auto-format)
- Service Type (dropdown, required)
  - Options: Fuel Systems, Car Wash, Environmental, General Inquiry
- Message (textarea, optional, max 500 characters)

**Form Validation:**
- Inline validation on blur
- Error messages in red below field
- Success state: green checkmark
- Submit disabled until all required fields valid

**Form Submission:**
- Loading spinner on button during submit
- Success: Modal confirmation "Thanks! We'll contact you within 4 hours"
- Error: Inline error message, keep form data
- Email confirmation sent to user

## Performance Considerations

### Initial Load
- Critical CSS inlined in `<head>` for above-the-fold content
- Defer non-critical JavaScript
- Lazy load images below fold
- WebP format with fallbacks
- Target: < 3 seconds initial load on 3G

### Scroll Performance
- CSS animations use `transform` and `opacity` (GPU-accelerated)
- Avoid layout thrashing (minimize reflows)
- Throttle scroll event listeners (debounce)
- Intersection Observer for scroll animations

### Image Optimization
- Responsive images with `srcset` and `sizes`
- Appropriate compression (JPG 80% quality)
- Modern formats (WebP with JPG fallback)
- Hero images optimized (< 200KB)
